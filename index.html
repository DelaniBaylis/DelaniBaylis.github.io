Databases

The artifact that I used for this is CS 320 Task assignment, that was made around 2020. What this file contains is that it contains a file that has a task that people will follow, and how the task is assigned to in the file that is in the cloud database, which means it's public and everyone can see it. The pre-enhancement is the file using the task system will use hash tables for the task. The post-enhancement is the files being used without the hash tables, and also simplifying the whole thing.

The project that I chose was one of the biggest examples of the cloud database, and the reason why some of the other databases can be better. The language that is used inside the database is java. What the cloud database helps with is the fact that it is open, and can be used by multiple people, and shared without reason, and edited with no problem. I show this in the enhanced version of this project. Whereas some databases are more secure, the openness of a cloud database can help a lot. The cons that come with it is security as well, since the database is not that secure it is really easy to change, unless there is a secure key on the files.

To me I feel like I did complete all the things that I wanted to show in this artifact about databases. I wanted to show what is the big thing that makes using a cloud database good, and what are the flaws of using a cloud database.

When I was making my changes I learned a lot of how cloud databases work. Since the files are saved in my cloud databases, I was able to just log in anywhere and make the necessary edits anywhere, showing how versatile a cloud database can be. I also learned that making these enchantments to the file made me see that codes can always be simplified down, to make it easier to edit later.
